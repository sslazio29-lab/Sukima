# CLAUDE.md（不変ルール・150行未満厳守）

> このファイルは**改変不可**。変更が必要なら依頼者の明示的承認を得てから。
> 全エージェント（Codex / Claude Code）は**毎回これを読んでから**作業を始める。

## 1. 作業規律

- 小タスクごとに **コミット・プッシュ**。`/clear` 推奨のキリで必ず行う。
- 作業内容（失敗含む）を毎回 **`WORKLOG.md`** に記録する。
- 大タスク終了時に各 md の肥大化を点検し、**コンテキスト節約のため最適化**する。
- 設計書・仕様（`SPEC.md` 等）の編集は **依頼者の承認を得てから**。`CLAUDE.md` 本体は不可。
- 再利用できる知見は **skills / hooks** 化してよい（自律的成長を歓迎）。
- 整合性チェックやタスク漏れ防止に **サブエージェントの並列作業を許可**。
- `CLAUDE.md` と `AGENTS.md` が矛盾したら **`CLAUDE.md` を優先**。

## 2. プロダクトの歯止め（最重要）

- **やることは3択クイズ1つだけ**。解答後に〇×と解説が出る。それ以外を作らない。
- **やり込ませる機能を作らない**：学習時間記録・成績グラフ・連続日数・ランキング・
  点数集計・タイマーは一切入れない。受験生の勉強時間配分を変えないための歯止め。
- Sukima は自宅学習の **「スキマの補助」に徹する**。
- 音声は別プロジェクト **Voice Study** の担当。Sukima には載せない。フォルダ・記録も混ぜない。
- 迷ったら「足す」より「足さない」。

## 3. 技術の固定方針

- **GitHub Pages の静的アプリで完結**。Cloudflare・Workers・KV・D1・R2・
  サーバー処理を**持ち込まない**。判定・解説表示・回答済記録はすべて iPhone のブラウザ内で完結。
- **リポジトリは public**（無料）。private は無料個人プランでは Pages 不可、かつ有料化しても
  配信される問題JSONはURL所持者から見えるため、コストに見合う秘匿効果がない。
- 回答済の記録は **localStorage**（数KB）。サーバー共有しない＝端末ごとに完全独立。
  他者が同じURLで解いても受験生の進捗には一切影響しない。

## 4. データと問題追加（最優先価値）

- **1ファイル＝1単元**の問題セット（JSON）。ファイル名は **`科目_単元.json`**（例：`日本史_室町時代.json`）。
  連番は使わない（Voice Study 用の番号であり Sukima とは無関係）。
- ファイル中身は**問題オブジェクトの配列**。科目・単元はファイル名が唯一の正とし、中身に書かない。
- 1問の構造：`問題文 / 選択肢3つ / 正解(1〜3の番号) / 解説`。難易度フィールドは持たない。
- **id は明示しない。配列の並び順そのものが id**。問題数＝配列長で一目で分かる。
- **追加は配列の末尾へ**。末尾追加なら既存の位置がズレず進捗が壊れない。
  途中挿入・並べ替えは進捗をズラすので避ける。
- **依頼者に目次を手書きさせない**。`manifest.json` は **GitHub Actions が push 時に自動生成**し、
  ファイル名規則の検証も同時に行う。規則違反・壊れたJSONは Action が赤く落ちて気づける
  （無言の失敗を作らない）。
- 問題追加が面倒になる作りにしない。「JSONを末尾に足して push するだけ」を死守。

## 5. 出題ロジック（守る）

- 各問は **未回答／回答済** の2状態。出題は**未回答プールから引く**。
- **正解 → 回答済になり、もう出ない**。**不正解 → 未回答のまま残り、また引かれる**。
  これだけで「苦手な問題が自然にしつこく出る」を実現。**復習用の独立画面は作らない**。
- 出題順は**素直にシャッフル**。「最低何問あける」等の規則は足さない（複雑化回避）。
- **リセットは単元単位**。ある単元を全問回答済にしたらその単元だけ未回答へ戻せる。
  他単元・他科目の進捗には触れない。localStorage キーは `科目/単元/問番号` の粒度。

## 6. 画面（極限までシンプル）

1. 科目を選ぶ → 単元を選ぶ
2. 問題数を選ぶ（受験生が毎回決める。固定しない）
3. 問題が出る → 選択肢3つをタップ
4. 即座に〇× → 解説 → 次へ
5. （その単元が全問回答済なら）リセットできる

凝った採点・集計・演出は作らない。「サクサク次へ」のテンポだけを追求する。

## 7. 作問担当の区別

- **数学** … 受験生が実際に間違えた問題を依頼者が改題したオリジナル（解法・方針を3択で選ばせる型）。
- **暗記科目・英単語** … **Claude が量産・配信してよい**領域。
